# Промпт для Gemma 4 / Gemini 2.5 Pro — сайт клана Arasaka

---

## Задача

Создай два отдельных HTML-файла: **index.html** (главная страница) и **wiki.html** (вики клана). Каждый файл должен быть полностью самодостаточным — весь CSS и JS внутри одного файла, никаких внешних зависимостей кроме Google Fonts.

---

## Дизайн и стиль

- **Общая эстетика:** тёмный промышленный стиль, вдохновлён игрой Satisfactory. Не киберпанк — именно индустриальный, утилитарный. Похож по структуре на сайт pepeland.net
- **Цветовая палитра:**
  - Фон глубокий: `#141414`
  - Фон панелей: `#191919`, `#1e1e1e`, `#222222`
  - Граница: `#2e2e2e` / `#333333`
  - Акцент (красный): `#B43535`
  - Текст основной: `#e8e8e0`
  - Текст приглушённый: `#888880`
  - Текст тусклый: `#444440`
- **Шрифты** (подключить через Google Fonts):
  - Заголовки, логотип, названия секций: `Rajdhani` (wght 400;500;600;700)
  - Основной текст, описания, параграфы: `Rubik` (wght 400;500;600)
  - Монospace метки, breadcrumbs, теги: `JetBrains Mono` (wght 400;500)
- **Важно по шрифтам:** Rajdhani не имеет кириллицы. Поэтому:
  - Всё что содержит русский текст (параграфы, описания, пункты списков) — только `Rubik`
  - Заголовки h1/h2/h3 внутри контента вики — тоже `Rubik`, не Rajdhani
  - Rajdhani только для: логотипа ARASAKA, названий секций (О клане, Навигация и т.д.), кнопок
  - Никогда не применяй `text-transform: uppercase` и `letter-spacing` к русскому тексту — это ломает рендеринг
- **Скруглённые углы** везде: кнопки, карточки, панели — `border-radius: 10px`
- **Фоновая сетка:** фиксированный паттерн из тонких линий цвета rgba(180,53,53,0.035) с шагом 48px
- **Scanline overlay:** фиксированный псевдоэлемент body::before с полосами rgba(0,0,0,0.03)

Заголовок: 🏯 Устав клана Arasaka

## I. Общие положения
1. Правила регулируют порядок взаимодействия и внутреннюю структуру клана Arasaka.
2. Их соблюдение обязательно для всех участников без исключений.
3. Присоединяясь к клану, игрок подтверждает согласие с данными правилами.
4. **Никнейм в игре должен полностью совпадать с никнеймом в Discord.** Пример: `mitsuk_i | макнагец`
5. Участник любой другой организации на серверах, где клан ведёт деятельность, не имеет права вступать в клан — исключения только по решению руководства.

## II. Правила поведения и коммуникации

### 2.1. Коммуникация
1. Токсичное, оскорбительное или провокационное поведение запрещено.
2. Публичные конфликты запрещены. Все споры — через руководство или Совет.
3. Запрещены кражи, гриф и обман соклановцев.

### 2.2. Клановый склад
1. Все участники обязаны пополнять склад, выполняя задания.
2. Ресурсы используются исключительно в интересах клана.
3. Продажа или передача ресурсов третьим лицам запрещена.
4. **Крупные личные склады запрещены.**
5. Новички не имеют доступа к складу без поручений.
6. Личные сундуки допускаются только в специально выделенной зоне.

### 2.3. Постройки и территория
1. Главный архитектор отвечает за планирование и участки.
2. Каждый участник обязан иметь жильё на территории клана.
3. Все постройки считаются собственностью клана.
4. Снос и перестройка — только с разрешения Главы или Совета.
5. Все постройки должны соответствовать стилю города.

## Важные технические требования

1. **Favicon:** добавить `<link rel="icon">` — можно SVG data URI с буквой A красного цвета
2. **Мобильная адаптация:** на экранах < 768px сайдбар и оглавление в вики скрываются, сетка карточек переходит в 1 колонку
3. **Анимация появления:** `.reveal { opacity:0; transform:translateY(16px); transition: opacity 0.6s, transform 0.6s }` + `.visible { opacity:1; transform:none }` — IntersectionObserver
4. **Бегущая строка:** `animation: ticker 32s linear infinite` — `@keyframes ticker { 0%{transform:translateX(100%)} 100%{transform:translateX(-100%)} }`
5. **Зелёная точка в статус-баре:** `animation: pulse 2s infinite` — мигание opacity
6. Никаких внешних библиотек кроме Google Fonts
7. Оба файла должны ссылаться друг на друга и на Discord корректно
